What is Books.AI?


Books.AI is designed to demonstrate the capability of OpenAI LLMs to summarise complex literary concepts quickly and concisely. The current build uses gpt-3.5-turbo, but gpt-4 is available for use via the OpenAI API.

The intention of this application is to demonstrate how, with a relatively simple back-end and some smart UI, a single developer with access to OpenAI API can build a useful and entertaining application powered by LLMs.

Prompts


The key to obtaining valuable and structured responses from an LLM is in submitting the correct prompts.

Listed below are some examples of the prompts used in this application to provide responses for the various parts of the results page.

To ascertain whether the user input is likely to be a book or not (cut off value set at 0.7):

"Return a single float value that represents your opinion on whether {self.input} is a book. Only return this float, no other text"

To return a JSON style dataset of book information,:

"Tell me the author, authors nationality, date of completion, and genre of the book {input}. Present it as a python dictionary with the author, authors nationality, date of completion, and genre of the book as keys and answers as values. Only output the raw python object in the format of:"

{ "author": "author value", "authors nationality": "nationality value", "date of completion": "date of completion value", "genre": "genre value" }

To return paragraph summaries on book main themes and writing styles:

"Write a paragraph summarising the key themes of the book {self.input}"

"Write a paragraph summarising the writing style of the book {self.input}"
